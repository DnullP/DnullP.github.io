    CPU Scheduling

    We will discuss the CPU Scheduling method by some steps:

    • ordinary scheduling
    • multicore scheduling
    • real-time scheduling

    When we talk about a scheduling method, we will meet with the terminology preempt

    If a process can be stopped by the system, and give the CPU to ohter process, we call the scheduling method as preemptive scheduling. If only when the process yields the CPU, the system can give the CPU to other process, we call the scheduling method as non-preemptive scheduling.

    1. Criteria for Scheduling

    • CPU utilization
    • Throughput: the number of processes that per time unit.
    • Turnaround time: the time from the arrival to the finish.
    • Waiting time
    • Response time (this is usually used for the interactive system)

    2. ordinary scheduling

    2.1. FCFS (first come first served)

    This method will cause the convoy effect.

    2.2. SJF (the shortest job first)

    However, we can't know the length of the job exactly. So we have an estimation method to do this.

    • Exponential average: γn+1=αγn+(1−α)tn+1\gamma_{n+1} = \alpha \gamma_n + (1-\alpha)t_{n+1}γn+1​=αγn​+(1−α)tn+1​

    We utilize the formula to calculate the estimation.

    The tn+1t_{n+1}tn+1​ is the length the most recent job.

    If we allow a process which has a short remained time to preempt a process which has a long remained time, we call this method as shortest remaining time first.

    2.3. Round-Robin Scheduling

    This method is a preemptive scheduling method.

    We set a time slice for each process. If the process can't finish in the time slice, the process will be preempted.

    And all the process will form a circle.

    2.4. Priority Scheduling

    This way gives each process a priority number. When a process which has a higher priority number comes, one of the runing process with lower priority number will be preempted.

    This method will cause the starvation problem. So we can aging the priority number to solve this problem, which means we need to boost the priority.

    2.5. Multilevel Queue Scheduling

    This is a composition of the previous scheduling methods.

    We divide the processes into different queues. Each queue has a priority number and the corresponding process will be scheduled to the CPU.

    Each queue has its own scheduling method.

    2.6. Multilevel Feedback Queue Scheduling

    This method just utilize the aging above the multilevel queue scheduling.

    Under the aging mechanism, the process is able to move between the queues.

    This method need four key define:

    • number of queues
    • scheduling algorithm for each queue
    • method used to determine when to upgrade or down grade of process
    • how to determine which queue to enter when process needs service

    Thread Scheduling

    We have said that the thread is the basic unit of the CPU scheduling rather than the process. But we need deeply discuss the thread scheduling especially.

    When we use the thread, usually it is the user mode thread. Thus it's necessary to dispatch the user thread to the kernel thread.

    That causes a Process contention scope problem: many user thread need to dispatch to the same kernel thread.

    The abstraction of the kernel thread orient the user thread is the lightweight process.

    And the LWP is scheduled by the OS kernel.

    A LWP is usually bound to a kernel thread.

    Multi-processors Scheduling

    • Multicores CPUs
    • Multithreaded cores
    • NUMA system
    • Heterogeneous multiprocessors
    • asymmetric multiprocessing
    • symmetric multiprocessing (SMP)

    A CPU has a lot of cores, and each core can process usually two thread. This is called Core Multithread (CMT), also called Simultaneous Multithreading (SMT) or hyper threading

    When we discuss the "cpu" in scheduling, it's usually the hardware thread.

    3. Why do we need the two hardware threads?

    We know that the writing and reading of memory is slow, and under the instruction level, we can caculate one thread when another is writing the data back to the memory.

    Besides, we can adjust the grain of the thread switching. If we run a lot of instruction of a thread, it will be easy to implement. But it's costly when you switch the thread because you need to reload all the instruction on the pipeline.

    It will be nice if we switch the thread per instruction circle so that the instruction of two threads can be executed alternately. As you might think, it will need a futher design to support.

    4. Loading Balance

    I am glad to write another article to discuss the usual loading balance algorithm.

    But now, just let us disscuss the part of it.

    4.1. Push and Pull Migration

    When a CPU is idle, it will like to pull some task from other CPU. And when a CPU is busy, it will like to push some task to other CPU. This two method cooperate to achieve the loading balance.

    4.2. Processor affinity

    We use cache in the CPU to speed up the fetching data from memory. When a process is running on the same CPU, the cache is hit more frequently. But when the process is migrate to another CPU, the cache will not be used. This phenomenon is called cache affinity.

    5. comparison

    This two factors to improve the CPU utilization is mutually exclusive.

    6. heterogeneous multiprocessors(HMP)

    This is related to the hardware architecture. Different core has different performance. So we can schedule the high performance-needed task to the high performance core and the low performance-needed task to the low one.

    A class example is the big.LITTLE architecture of arm.

    Real-Time Scheduling

    When we use some software, we want it to response in a short time. To get this, we cannot let the process to wait for too long.

    There is two ways to do the work:

    • soft real-time: the system can miss the deadline sometimes, it just try to make the latency minimum.
    • hard real-time: the system give a guarantee that the process can get a response in a certain time.

    But before we discuss the real-time scheduling algorithm, let's explain the response time

    7. Response Time

    Conflict is the situation that we need to scheule a process right now but no vacancy is remain, so we need to make a process yield the CPU.

    Dispatch is the time to change every infomation which needed to be changed and then switch to the target process.

    8. Rate Monotonic Scheduling

    To get the real-time scheduling, we must give each process a priority number, so that one can preempt another when it's time to run.

    10. Proportional Share Scheduling

    This is a kind of algorithm based on the fairness.

    We must make sure the running time of each process is proportional to the share of the process.

    There are many methods to achieve this, like the lottery scheduling and stride scheduling.
